Information about RChain Platform and Architecture:

RChain is an open source, emerging blockchain platform with a smart contract language called Rholang. The syntax and execution mechanism are derived directly from the rho-calculus, a variant of Milner’s π-calculus. The platform is intended to support correct-by-construction smart contracts with behavioral types. It is not possible to have bytecode injection attacks.  RChain is the first implementation of the Correct By Construction (CBC) Casper consensus algorithm.

Being derived from the rho-calculus, Rholang is a language in the process calculi family of languages.  Rholang is an asynchronous message based concurrent language with reflection, thus supporting higher-order mobility, like the higher-order π-calculus, but with far simpler syntax and semantics. Rholang has deep mathematical foundations making it suitable for automatic formal verification of code. In particular, Rholang enjoys a behavioral type system that has already been shown to be of crucial value in the blockchain application space.

When considered in the context of RSpace, its execution mechanism and a unique key-value database supporting the storage of both data and active code, Rholang may also be seen as a next generation query language that reconciles relational query mechanisms with nosql. This means that as a smart contracting language Rholang is ideal for making blockchain a platform for data intensive applications where data is stored on chain, as well as a process modeling language, such as XLang, BEPL, or BPML.

Rholang and RChain core platform examples are available for developers to follow here. 

There’re APIs available for the platform that allow a person skilled in any language to develop apps.

The core RChain platform is developed in Scala with extensive use of the cats library for pure, typeful functional abstractions. Those desiring core platform development internships, should preferably understand the structure and logic of the cats library and the best practices in using it.  https://github.com/typelevel/cats  and https://typelevel.org/cats/Also required for core platform internships: familiarity with and/or interest in Test frameworks, generative and law based testing using e.g. ScalaTest, Discipline and Test Driven Development.  However, Scala expertise is not a requirement for participation in the hackathon.

Registration and Selection Process:

Using the registration form, candidate hackathon participants need to complete registration of the teams and component individuals including (a) generating and submitting a REV address for EACH CANDIDATE using one of the interfaces to the RChain network and (b) providing a github account for EACH CANDIDATE. The REV address will be used for rewards, prizes etc. Github familiarity (how Github works, making repositories, forks and feature branches, creating pull requests etc.) is required of ALLCANDIDATES.  The second part of the screening / selection process involves using Github to create pull requests.

  1. Register team and individuals (all individuals in the team use the same team name) using the same email as the one used for the respective github accounts.  Teams also have the ability to describe a problem statement of their choice if they have a particular one they want to work on. Several problem statements will be made available by RChain for teams that do not have a self-created problem statement. Each team member can also provide a cell phone number and the corresponding SMS email address (typically phonenumber@mail.provider.com or some such format) to receive SMS notifications and reminders.
  2. All individuals participate in the online qualification test that includes questions in Scala, JavaScript, Python, Rholang and Cryptography + Blockchain concepts.
  3. Qualifying candidates will participate in a preliminary contest to create pull requests on github for smallish specified technical issues
  4. Hackathon participants will be selected based on their weighted performance in steps 2 and 3.
  5. Teams will be formed based on selected participants, keeping the problem statements in mind.
  6. Teams will select and finalize scope of the problem statements they will work on.
  7. Actual Hackathon Oct 10-11, 2020
  8. Judgement of teams by judges
  9. Prizes
  10. Opportunity for select teams and/or individuals to participate with RChain community in project based internships lasting three months to a year+

We hope the participants have an exciting, fun and wonderful experience with the hackathon process and the event.
